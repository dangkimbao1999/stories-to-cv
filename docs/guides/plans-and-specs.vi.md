# Plans and Specs

## Khi nào cần spec

- feature mới
- đổi kiến trúc
- đổi behavior qua nhiều layer
- thay đổi liên quan privacy/security

## Khi nào cần plan

- task nhiều bước
- task có deploy/verify
- task có migration hoặc async workflow

## Nơi lưu

- spec: `docs/specs/`
- plan: `docs/plans/`
- runlog: `docs/runlogs/`

## Quy tắc

- ngắn, rõ, có verification
- không viết như tài liệu marketing
- update khi direction thay đổi lớn
