# Plans

`docs/plans/` is the execution memory of the project.

Use this folder to track non-trivial implementation plans so future work can start from previous decisions instead of starting from zero again.

## Rules

- Before starting a new feature or a multi-step refactor, read related plans here first.
- If a new task continues or changes a previous direction, reference the older plan in the new one.
- Plans should be written in English.
- Keep plans short and execution-focused.
- When direction changes materially, update the plan or add a new superseding plan instead of silently drifting.
- Follow [docs/guides/plan-writing-for-codex.en.md](C:\Users\dangk\Documents\stories-to-cv\docs\guides\plan-writing-for-codex.en.md) when writing new plans.

## Naming

Use:

- `YYYY-MM-DD-<topic>.md`

Examples:

- `2026-06-11-kb-schema-foundation.md`
- `2026-06-11-cv-generation-v1.md`

## Expected structure

Start from [TEMPLATE.md](C:\Users\dangk\Documents\stories-to-cv\docs\plans\TEMPLATE.md).

Typical contents:

- goal
- scope
- risks
- execution steps
- verification
- links to prior relevant plans when applicable
