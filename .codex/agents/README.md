# Codex Agents

These files describe focused sub-agent roles for common workflows in this repo.

- `test-police.md`: protects TDD discipline and unit/integration quality
- `e2e-police.md`: owns Playwright coverage and browser-level validation
- `run-operator.md`: starts and monitors local services
- `log-watcher.md`: tails local and remote logs continuously
- `deployer.md`: performs guarded deploy and post-deploy verification
- `debugger.md`: drives systematic debugging across web, worker, DB, and provider layers
- `git-worktree-police.md`: enforces safe branch/worktree workflow
- `ci-police.md`: keeps CI green and actionable
- `memory-keeper.md`: records durable project knowledge
