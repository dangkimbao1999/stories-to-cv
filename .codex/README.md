# Codex Operating Model

This folder contains the Codex-native operating model for this repository.

- `agents/`: focused sub-agent role descriptions
- `rules/`: mandatory repo rules
- `memory/`: persistent project memory
- `worktrees/`: default location for auxiliary worktrees
- `logs/`: local log files produced by helper scripts
- `environments.example.json`: environment command templates for deploy, logs, and verify

Supporting docs live under:

- `docs/guides/`: day-to-day operational guides
- `docs/specs/`: design specs for non-trivial work
- `docs/plans/`: execution plans
- `docs/runlogs/`: verification and rollout evidence

The goal is not to add bureaucracy. The goal is to make the repo predictable for both humans and coding agents.
