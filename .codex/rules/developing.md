# Developing Rules

- Keep changes small and vertical.
- Move business logic into shared packages, not UI components.
- Prefer boring, readable modules over framework-heavy abstractions.
- Add code only when a test or user workflow justifies it.
- Do not introduce infrastructure that the current traffic profile does not need.
- Keep privacy and traceability visible in the code, not hidden in convention.
- For non-trivial work, create a short spec or plan before implementation.
