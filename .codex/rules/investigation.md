# Investigation Rules

- Start by locating the failing boundary: UI, API, worker, DB, or provider.
- Use logs, health checks, and persisted records before guessing.
- When investigating, collect:
  - symptom
  - environment
  - identifiers
  - first bad boundary
  - likely root cause
  - next verification step
- Turn resolved bugs into a regression test or a memory note when the lesson is reusable.
