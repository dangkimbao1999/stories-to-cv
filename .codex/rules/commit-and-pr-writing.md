# Commit And PR Writing Rules

- Commit messages should be conventional and behavior-oriented.
- Add a short commit body when the reason or constraint is not obvious from the subject alone.
- Draft PR summaries should follow this structure when relevant:
  - Summary
  - Why
  - Spec
  - Plan
  - Scope
  - Test plan
  - Verification
  - Risks / follow-ups
  - References
- Include exact verification commands when known.
- Explicitly say when a check was not run.
- Link related plans/specs/runlogs when they exist.
