# Planning Rules

- For work that spans multiple files or changes behavior across boundaries, write a short spec or plan first.
- Plans in this repository should be written in English.
- Specs belong in `docs/specs/`.
- Execution plans belong in `docs/plans/`.
- Verification evidence for risky changes belongs in `docs/runlogs/`.
- Before starting non-trivial feature work, review related existing plans in `docs/plans/`.
- New plans should reference prior plans when they continue, replace, or refine earlier work.
- Follow the plan structure guide in `docs/guides/plan-writing-for-codex.en.md`.
- Keep them lightweight: enough to guide work, not enough to become ceremony.
